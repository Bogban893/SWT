Тестовый сценарий — это последовательность шагов для проверки определённого аспекта системы.

### Структура тестового сценария:

1. **ID** — уникальный идентификатор
2. **Название** — описание того, что тестируется
3. **Предусловия** — состояние системы перед тестом
4. **Шаги** — последовательность действий
5. **Ожидаемый результат** — что должно произойти
6. **Фактический результат** — что произошло при выполнении
7. **Статус** — Pass/Fail
8. **Замечания** — если нужны дополнительные комментарии

### Пример:

| ID | TC-001 |
|---|---|
| **Название** | Пользователь может успешно войти в систему |
| **Предусловия** | Пользователь не авторизован, имеет корректные учётные данные |
| **Шаги** | 1. Открыть страницу логина; 2. Ввести username; 3. Ввести password; 4. Нажать кнопку Login |
| **Ожидаемый результат** | Пользователь переходит на главную страницу, видит своё имя в правом верхнем углу |
| **Фактический результат** | ✓ Совпадает с ожидаемым |
| **Статус** | Pass |

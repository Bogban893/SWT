Треугольник равновесия проекта (Project Triangle, Iron Triangle) — модель, показывающая основные ограничения проекта:

**Три стороны треугольника:**

1. **Scope (Область)**
   - Какую работу нужно выполнить
   - Функциональность продукта
   - Требования

2. **Time (Время)**
   - Сколько времени есть на выполнение
   - Сроки проекта

3. **Cost (Стоимость)**
   - Сколько денег выделено на проект
   - Бюджет проекта

**Принцип:** Если увеличить одну сторону, нужно изменить другие:
- Больше функциональности (↑ Scope) → больше времени или денег
- Меньше времени (↓ Time) → либо меньше функциональности, либо больше денег
- Меньше денег (↓ Cost) → либо меньше функциональности, либо больше времени

**Центр треугольника:** Качество — должно остаться неизменным, проекты не должны жертвовать качеством для соблюдения других ограничений.

### Жизненный цикл ошибки (Bug Lifecycle):

**Состояния:**

1. **New** — ошибка только что зарегистрирована
2. **Assigned** — ошибка назначена разработчику
3. **In Progress** — разработчик работает над ошибкой
4. **Fixed** — разработчик исправил ошибку
5. **Verified** — тестировщик проверил исправление (может быть несколько циклов)
6. **Closed** — ошибка закрыта
7. **Reopen** — если после проверки обнаружилось, что ошибка не исправлена

### Информация об ошибке:

- **ID** — уникальный идентификатор
- **Название** — краткое описание
- **Описание** — подробное описание проблемы
- **Приоритет** — как быстро нужно исправить (Critical, High, Medium, Low)
- **Severity** — насколько серьёзна проблема (Critical, Major, Minor)
- **Шаги для воспроизведения** — как повторить ошибку
- **Окружение** — на какой ОС, браузере и т.д.
- **Выполняемые тесты** — какие тесты не прошли
- **Назначена** — кому дана ошибка
- **Дата создания/закрытия** — временные метки

### Инструменты для отслеживания:

- **Jira** — основной инструмент в индустрии
- **Azure DevOps** — инструмент от Microsoft
- **Bugzilla** — открытое ПО для отслеживания
- **GitHub Issues** — встроенное отслеживание ошибок
- **Redmine** — открытое ПО для управления проектами

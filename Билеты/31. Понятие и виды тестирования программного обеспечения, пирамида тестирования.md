### Виды тестирования по уровню:

**1. Модульное тестирование (Unit Testing)**
- Тестирование отдельных модулей/функций
- Делается разработчиками
- Быстрое и дешёвое

**2. Интеграционное тестирование (Integration Testing)**
- Тестирование взаимодействия модулей
- Проверка, что компоненты правильно работают вместе

**3. Системное тестирование (System Testing)**
- Тестирование всей системы целиком
- Проверка соответствия требованиям

**4. Приёмочное тестирование (Acceptance Testing)**
- Тестирование заказчиком
- Проверка, что система соответствует бизнес-требованиям

### Пирамида тестирования:

```
       Acceptance Tests
      /     UI Tests    \
    /   Integration Tests  \
  / Unit Tests (быстрые, дешёвые) \
```

- **Основание:** много Unit тестов (дешевые, быстрые)
- **Средняя часть:** интеграционные тесты
- **Вершина:** немного приёмочных тестов (дорогие, медленные)

### Виды тестирования по функциональности:

- **Functional Testing** — соответствие требованиям
- **Non-functional Testing** — производительность, безопасность, и т.д.
- **Regression Testing** — проверка, что изменения не сломали существующую функциональность
- **Performance Testing** — проверка скорости и стабильности
